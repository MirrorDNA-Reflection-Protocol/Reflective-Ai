# Contributing to Reflective AI

Thank you for your interest in Reflective AI™. This category is still emerging, and community input helps shape its evolution.

---

## How to Contribute

### **1. Feedback on Concepts**
If you have thoughts on the philosophical or architectural foundations of Reflective AI:
- Open an issue with the `concept` label
- Be specific about which principle you're addressing
- Provide reasoning or alternative perspectives

### **2. Implementation Examples**
If you've built your own reflective systems:
- Share your approach in an issue with the `implementation` label
- Describe what worked, what didn't
- Link to code or documentation if public

### **3. Technical Corrections**
If you spot errors in the architecture or technical descriptions:
- Open an issue with the `correction` label
- Cite specific sections
- Provide accurate information with sources

### **4. Extensions and Experiments**
If you've extended these ideas in new directions:
- Open an issue with the `experiment` label
- Describe your approach and results
- Share learnings

---

## What's Not Accepted

- Generic "this is cool" comments (use GitHub stars instead)
- Requests for code that doesn't exist yet
- Demands for support on unrelated projects
- Marketing or promotional content

---

## Code of Conduct

**Be respectful.** Ideas are challenged, people are not.  
**Be specific.** Vague feedback isn't actionable.  
**Be honest.** If something doesn't work, say so clearly.

---
